# セキュリティポリシー

## サポート対象

セキュリティ修正は原則として最新リリースを対象に提供します。問題を確認する前に、[最新リリース](https://github.com/hayashiii-ghub/shelfdrop/releases/latest)で再現するか確認してください。

## 脆弱性の報告

脆弱性や、ファイル操作による意図しないデータ消失の可能性を見つけた場合は、詳細を公開Issueへ投稿しないでください。

GitHubの`Security`タブにある`Report a vulnerability`から非公開で報告してください。Private vulnerability reportingを利用できない場合は、機密情報や再現コードを含めずにIssueを作成し、非公開の連絡方法が必要であることだけを伝えてください。

報告には次の情報を含めてください。

- 影響を受けるShelfDropのバージョン
- macOSのバージョンとMacの種類
- 問題の概要と想定される影響
- 最小限の再現手順
- 既知の回避策

受領後、内容を確認し、公開時期と修正方針を調整します。
